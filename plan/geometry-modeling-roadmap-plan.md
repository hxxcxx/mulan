# 几何编辑与建模能力后续计划

## 当前已完成

- 建立 `GeometryMutation` / `GeometryEditService`，曲线和面资产修改开始走统一提交层。
- `DocumentOperation` 已能提交几何资产 mutation，并接入 Undo / Redo。
- 曲线控制点拖动仍从现有 GripDragTool 进入，但底层已转为统一 mutation 提交。
- 编辑共享几何资产前会执行 make unique，避免复制实体共享资产时改一个影响多个。
- 当前支持的结构化 mutation：
  - 曲线元素替换。
  - 面定义替换。

## 后续工具入口

### 1. 曲线控制点编辑命令

- 选中控制点后删除控制点。
- 在曲线上插入控制点。
- 多选控制点整体移动。
- B 样条插入节点，而不是简单插入控制点。
- NURBS 权重编辑。
- 所有修改最终都生成新的曲线 primitive，并通过 `GeometryEditService` 提交。

### 2. 面编辑器

- 选中面顶点后拖动顶点。
- 面轮廓插入点、删除点。
- 支持洞和多轮廓。
- 面轮廓修改后重新清理 loop、验证自相交 / 重叠、重新三角剖分。
- 所有修改最终生成新的 `FaceDefinition`，通过 `GeometryEditService` 提交。

### 3. 子对象编辑

- 选择目标继续细化到点、边、面、控制点。
- 子对象高亮和实体高亮分层。
- 子对象夹点和 Gizmo 复用同一套拖动提交管线。
- 子对象编辑需要区分修改实体 transform 和修改 geometry asset。

## 后续建模能力

### 1. 面到体

- 面拉伸成体。
- 面沿路径扫掠。
- 曲线旋转成体。
- 后续布尔先预留入口，不急着做完整布尔。

### 2. 体编辑

- 面拉伸 / 偏移。
- 边倒角 / 圆角。
- 点移动。
- 面、边、点修改仍然要落到统一 mutation / operation 层。

### 3. Gizmo 与精确输入

- 建立统一平移 / 旋转 / 缩放 Gizmo。
- 支持轴约束、平面约束、snap、work plane。
- 支持距离、角度、比例等数值输入。
- Gizmo hit test 使用独立身份，不污染普通几何选择。

### 4. 参数化历史雏形

- 记录 Sketch -> Extrude -> Body 这类操作来源。
- 修改草图后能重建关联体。
- 先做轻量历史，不直接追求完整参数 CAD。

## 实施顺序建议

1. 面顶点编辑和面轮廓重建。
2. Extrude 拉伸成体。
3. Gizmo 精确变换。
4. 子对象精确高亮。
5. 曲线插点 / 删除点工具入口。
6. 参数化历史雏形。

## 暂存原因

本次代码已经完成几何资产修改提交层。上面这些属于工具能力、建模能力和交互能力扩展，范围更大，应该分步骤推进，避免把提交层和具体建模命令混在一起。
